Распишите свое понимание пространства имен Пространство имен, это инструмент, который появился в PHP 5.3 помогающий избегать ошибок при дублировании функций, методов, классов после загрузки нескольких файлов .php .
То есть если в условных файлах file1.php и file2.php объявлена функция с одинаковым именем function getPrise(){ ... } То при загрузке файлов include file1.php; include file2.php произойдет ошибка, т.к код приналежит глобальному пространству имен
Чтобы это не происходило можно объявить собственное пространство имен использую ключевое слово namespace, указывая пространство имен нашего файла отдельно.
Исходя из этого, чтобы вызвать функцию function getPrise(){ ... } нужно указать пространство, в котором она объявлена namespace file1 или namespace file2 (прописывается самом вверху кода)
Распишите свое понимание исключений (Exception) и зачем они нужны Исключения это инструмент PHP, созданный для удобной обработки ошибок. Например нам нужно вывести данный из файла, но файла может и не быть, произойдет ошибка. В качестве решения проблемы были придуманы исключения. Если в функции произошла какая-то ошибка и она не может выполнить свою работу, то она выбрасывает исключение.
Например:
if (!file_exists($file)) {throw new Exception("Ошибка: файл $file не существует");}
